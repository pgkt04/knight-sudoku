# Knight Sudoku

A cross-platform Sudoku game with an anti-knight constraint. Runs on browser, iOS, and iPadOS.

## Rules

Standard 9x9 Sudoku rules, plus: cells separated by a chess knight's move (L-shape) cannot contain the same digit.

## Implementation Phases

### Phase 1: Game Engine (pure TypeScript, no UI)

All logic is pure functions with no React dependency — testable in isolation.

1. **Types** — `Board` (9x9 grid), `Cell` (value, pencil marks, given vs player-entered), `Difficulty` enum
2. **Knight-move calculation** — given a cell `(r, c)`, return all cells reachable by a knight's L-shaped move, filtered to stay within 9x9 bounds
3. **Validator** — check if placing digit `d` at `(r, c)` violates:
   - Row constraint
   - Column constraint
   - 3x3 box constraint
   - Knight constraint (the unique part)
4. **Solver** — backtracking solver that respects all four constraints. Needed for puzzle generation and to verify unique solutions.
5. **Generator** —
   - Fill a complete valid board (backtracking with randomization)
   - Remove cells one at a time, checking that the puzzle still has a unique solution
   - Difficulty rating based on: number of givens, required solving techniques, backtracking depth
6. **Pre-built puzzles** — JSON files with curated puzzles for each difficulty, for instant play without generation delay

### Phase 2: React UI (browser)

7. **Board + Cell rendering** — CSS Grid layout, highlight row/col/box/knight-neighbors of selected cell
8. **Number input** — NumberPad component, keyboard support (1-9 keys), tap on mobile
9. **Pencil marks** — toggle pencil mode, display small numbers in cells
10. **Error highlighting** — highlight conflicts in real-time (same digit in row/col/box/knight-reach)
11. **Undo/Redo** — `useReducer` with action history stack
12. **Timer** — counts up, pauses when app is backgrounded
13. **Game flow** — difficulty selection → play → win detection → completion screen
14. **Persistence** — auto-save game state to `localStorage`, resume on return

### Phase 3: Polish

15. **Responsive layout** — works on phone (portrait), tablet (portrait + landscape), desktop browser
16. **Touch optimization** — tap cell → tap number, large touch targets
17. **Knight-move visualization** — optionally highlight which cells are a knight's move from the selected cell (key UX feature)
18. **Animations** — subtle cell fill animation, win celebration

### Phase 4: React Native + iOS

19. **Create RN project** — separate project, imports `src/game/` as shared package
20. **Rebuild UI in RN** — `<View>`, `<Text>`, `<TouchableOpacity>` equivalents of web components
21. **Safe areas** — handle iPhone notch/Dynamic Island, iPad multitasking
22. **App icon + splash screen** — required for App Store
23. **Test on simulator** — iPhone and iPad layouts
24. **Test on real device** — needs developer account to be active
25. **App Store submission** — screenshots, description, review

## Key Design Decisions

### Undo/Redo
Game state reducer stores a history stack. Every move pushes the previous state onto the undo stack.

### Puzzle Generation Performance
Generating a knight-sudoku with a unique solution can be slow. Generate puzzles in a Web Worker so the UI doesn't freeze. Have pre-built puzzles available for instant play while a fresh one generates in the background.

### Knight-Move Visualization
Key differentiator from regular Sudoku apps. When a cell is selected, all knight-reachable cells should be subtly highlighted so the player can see the constraint visually.

### Difficulty Levels
Four levels: Easy, Medium, Hard, Expert. Difficulty is controlled by number of givens and complexity of required solving techniques.

### Monetization
Paid app on the App Store. Free on the web (browser version).
